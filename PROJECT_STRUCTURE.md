# Signal Equalizer – Project Structure

## Project statement

**Signal Equalizer** is a web app for editing audio in frequency and wavelet domains. The user loads a signal, adjusts gains per band or per component via sliders, and hears/sees the result.

- **Generic mode** (later): frequency-only, no wavelets. Sliders and all graphs are frequency-based (FFT, spectrogram, audiogram). Sliders are loaded from a user schema file (e.g. frequency ranges 10–100 Hz, 400–1000 Hz).
- **Custom modes** (Human first, then Animal, Medical, Music): each has **frequency domain** (FFT + sliders by frequency ranges) and **optimal wavelet domain** (wavelet plot + sliders by wavelet components). Human mode uses e.g. Haar as the optimal wavelet for human voices; other modes use their own wavelet. Each custom mode also has AI comparison (later).
- We work on **Human mode** only for now; other modes are added over time.

## UI layout (Human mode – professional structure)

Layout order follows: **header** → **body** (left **sidebar** + right **main content**).

1. **Header**
   - Left: application title + current file name.
   - Right: **Change Mode** | **Upload** | **Export**.

2. **Sidebar (left)**
   - Mode selector: e.g. **Human mode** (only for now).
   - **Frequency domain equalizer**: section title + sliders controlling the 4 voices by frequency ranges.
   - **Wavelet domain equalizer**: section title + sliders controlling the 4 voices by optimal (e.g. Haar) wavelet components.

3. **Main content (right)**
   - **Row 1 – Signals**: two linked cine viewers: **Input Signal** | **Output Signal** (waveform + shared transport: play/pause/stop/speed/zoom).
   - **Row 2 – Frequency spectrum**: section title **Frequency Spectrum (FFT)** + **Audiogram scale** toggle. Two cards: **Input FFT** | **Output FFT**.
   - **Row 3 – Optimal wavelet domain**: section title + two cards: **Input wavelet** | **Output wavelet** (or one combined chart).
   - **Row 4 – Spectrograms**: toggle show/hide, then **Input spectrogram** | **Output spectrogram**.

Colors and styling can differ; the important part is this **element order and grouping** (header, sidebar controls, main: signals → FFT → wavelet → spectrograms).

## What to write in every file (simple structure)

Use this as a one- or two-line **purpose** at the top of each file (comment or docstring).

| File | Statement (what this file does) |
|------|--------------------------------|
| **Backend** | |
| `app.py` | Flask app: CORS, `/upload`, `/process`, `/save_schema`, `/load_schema`, `/sample/human`. No UI. |
| `dsp.py` | DSP only: FFT, spectrogram, frequency equalization, wavelet decomposition and wavelet equalization. |
| `modes.py` | Returns mode config (name, slider labels, freq_bands, wavelet, wavelet_levels) for Human (and later other modes). |
| `schemas/*.json` | User-editable slider config (frequency ranges, labels) for loading into the app. |
| `generate_human_sample.py` | One-off script: build synthetic 4-voice mix WAV and save to `data/human_sample.wav`. |
| **Frontend** | |
| `main.jsx` | Entry: render React App into root. |
| `App.jsx` | State and layout: header (title, file, Upload, Export, Change Mode), sidebar (mode, freq sliders, wavelet sliders), main (input/output signals, FFT, wavelet, spectrograms). |
| `api.js` | HTTP client: upload audio, process (mode + sliders), save/load schema. Base URL backend. |
| `index.css` | Global theme and layout: variables, header, sidebar, main grid, cards, sliders, charts. |
| `AudioUploader.jsx` | File input; on select calls parent `onFileSelect(file)`. |
| `ModeSelector.jsx` | Buttons to switch mode (Human only for now). |
| `WaveformViewer.jsx` | Canvas: draw time-domain waveform; optional playhead; linked view via shared props. |
| `SliderGroup.jsx` | N sliders (0–2), labels, values, `onChange(array)`. |
| `FFTChart.jsx` | One FFT plot: freq vs magnitude; `variant` input/output for color; supports audiogram scale. |
| `WaveletChart.jsx` | Bar chart of wavelet level energies (input and/or output). |
| `SpectrogramViewer.jsx` | Canvas heatmap from times, freqs, magnitudes. |
| `TransportControls.jsx` | Play, Pause, Stop, speed; callbacks to parent. |
| `AudiogramToggle.jsx` | Toggle linear vs audiogram scale for FFT; callback to parent. |

## Human mode only (current scope)

- **Frequency domain:** One set of sliders (Voice 1–4) with freq bands from `modes.py`; FFT and spectrogram for input and output.
- **Wavelet domain:** One set of sliders for the same 4 components via Haar wavelet; wavelet energy plot for input and output.
- **Output:** One equalized signal (freq EQ then wavelet EQ in sequence).
- **Data:** Use `data/human_sample.wav` from `generate_human_sample.py` for testing.

Later: Generic mode (schema-driven freq sliders, no wavelets); then Animal, Medical, Music modes with their own wavelet and schemas.
